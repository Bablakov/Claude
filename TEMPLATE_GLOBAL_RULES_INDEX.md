## 🎯 Общая философия

### Принцип 1: Четкость
Все правила должны быть понятны с первого прочтения. Нет двусмысленности.

### Принцип 2: Практичность
Правила должны решать реальные проблемы, не усложнять процесс.

### Принцип 3: Исключения допустимы
Правило = "обычно делай так", "исключения = когда не применяется"

### Принцип 4: Обновляемость
Правила живые. Если не работают → обновляем.

## 🔗 Как использовать GLOBAL_RULES

### Для разработчика/специалиста:
1. Читаешь NORMATIVE_DOC для своей задачи
2. В NORMATIVE_DOC ты видишь ссылки на GLOBAL_RULES файлы
3. Переходишь и читаешь общие правила
4. Применяешь общее + специфичное для платформы

**Пример:**
```
Читаю NORMATIVE_DOCS/reddit.md
Вижу ссылку: [Writing Standards](../../GLOBAL_RULES/writing_standards.md)
Переходу, читаю общие правила писания
Применяю их к Reddit (учитывая особенности платформы)
```

### Для Claude при создании чата:
1. Ты даешь мне ссылку на NORMATIVE_DOC
2. Я читаю NORMATIVE_DOC
3. Я видю ссылки на GLOBAL_RULES
4. Я слежу чтобы соблюдать оба набора правил

## 📝 Примечание для проектов

Каждый проект использует GLOBAL_RULES как база, но может иметь PROJECT_RULES которые переопределяют или расширяют эти правила.

Иерархия:
```
GLOBAL_RULES (база для всех)
    ↑
PROJECT_RULES (переопределение/расширение для конкретного проекта)
    ↑
NORMATIVE_DOCS (специфичные для тематики/платформы)
```
